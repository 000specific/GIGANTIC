# Contributing to GIGANTIC

Thank you for your interest in contributing to GIGANTIC!

---

## Ways to Contribute

- **Bug reports**: Open an issue describing the problem
- **Feature requests**: Suggest new functionality via issues
- **Documentation**: Help improve guides and tutorials
- **Code**: Submit pull requests for bug fixes or new features

## Getting Started

1. Fork the repository
2. Clone your fork locally
3. Create a branch for your changes
4. Make your changes following our conventions
5. Test your changes
6. Submit a pull request

## Code Style

Please follow the conventions documented in [conventions.md](conventions.md):

- Use the GIGANTIC variable naming patterns
- Include AI attribution headers in scripts
- Use self-documenting table headers
- Follow the directory structure standards

## Pull Request Process

1. Ensure your code follows GIGANTIC conventions
2. Update documentation if needed
3. Test with the demo dataset
4. Describe your changes clearly in the PR

## Reporting Issues

When reporting bugs, please include:

- GIGANTIC version
- Operating system
- Steps to reproduce
- Expected vs actual behavior
- Relevant log files

## Questions?

Open an issue with your question and we'll do our best to help.

---

*We appreciate all contributions to GIGANTIC!*
